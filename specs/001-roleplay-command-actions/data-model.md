# Data Model: Chat and Roleplay Command Actions

## 1. CommandAction

Represents a manual user submission from the composer.

Fields:
- `ActionId` (string, required): unique identifier.
- `SessionId` (string, required): target roleplay session.
- `Intent` (enum, required): `Instruction | Message | NarrativeByCharacter`.
- `PromptText` (string, required): trimmed input text.
- `SelectedCharacterId` (string, optional): required when intent is character-scoped.
- `SelectedCharacterType` (enum, optional): predefined/custom classification.
- `SubmittedVia` (enum, required): `PlusButton | SendButton`.
- `SubmittedAtUtc` (datetime, required).

Validation rules:
- `PromptText.Trim().Length > 0`.
- `Intent=Instruction` -> `SelectedCharacterId` must be null/ignored.
- `Intent=Message|NarrativeByCharacter` -> `SelectedCharacterId` is required and must reference an available identity.

State transitions:
- `Draft -> Validated -> Routed -> Completed`.
- `Draft -> ValidationFailed`.
- `Routed -> ExecutionFailed`.

## 2. CharacterSelection

Represents currently selected identity for character-scoped actions.

Fields:
- `CharacterId` (string, required).
- `DisplayName` (string, required).
- `SourceType` (enum, required): `Persona | SceneCharacter | CustomCharacter | Other`.
- `IsAvailable` (bool, required).
- `AvailabilityReason` (string, optional).

Validation rules:
- `CharacterId` must be unique in rendered options.
- `CustomCharacter` requires non-empty custom display value before character-scoped submit.

State transitions:
- `Unselected -> Selected`.
- `Selected -> Cleared`.
- `Selected -> Replaced`.

## 3. ContinueAsSelection

Represents Continue As popup state at time of Continue or Clear.

Fields:
- `SessionId` (string, required).
- `SelectedParticipants` (list enum, required): subset of `You | Npc | Custom`.
- `IncludeNarrative` (bool, required).
- `SelectionOrder` (list enum, required): deterministic participant order snapshot.
- `TriggeredAction` (enum, required): `Continue | Clear`.
- `TriggeredAtUtc` (datetime, required).

Validation rules:
- `TriggeredAction=Continue` requires at least one selected participant OR `IncludeNarrative=true`.
- All selected participants must be currently available for session/mode.
- `SelectionOrder` must not contain duplicates.

State transitions:
- `Idle -> Updated` (toggle/select operations).
- `Updated -> Cleared` (Clear action unselects all).
- `Updated -> Executed` (Continue action produces outputs).

## 4. ContinuationBatchResult

Represents outcome generated by a Continue operation.

Fields:
- `BatchId` (string, required).
- `SessionId` (string, required).
- `ParticipantOutputs` (list, required): one output entry per selected participant in deterministic order.
- `NarrativeOutput` (string, optional): non-character scene advancement text.
- `Succeeded` (bool, required).
- `FailureReason` (string, optional).

Validation rules:
- If `Succeeded=true`, output count must equal selected participant count.
- `NarrativeOutput` can exist only when `IncludeNarrative=true`.

## 5. ConversationTurnEvent

Audit/event record for command and continue operations.

Fields:
- `EventId` (string, required).
- `SessionId` (string, required).
- `EventType` (enum, required): `CommandSubmitted | ContinueExecuted | ContinueCleared | ValidationError | ProcessingError`.
- `Intent` (enum, optional): for command events.
- `ParticipantScope` (string array, optional): for continue events.
- `OutcomeStatus` (enum, required): `Success | ValidationFailed | Failed`.
- `CreatedAtUtc` (datetime, required).
- `DisplayContent` (string, optional): visible timeline content (includes instruction events).

Relationships:
- `RolePlaySession 1 -> many CommandAction`.
- `RolePlaySession 1 -> many ContinueAsSelection`.
- `ContinueAsSelection 1 -> 1 ContinuationBatchResult` when action is Continue.
- `RolePlaySession 1 -> many ConversationTurnEvent`.
